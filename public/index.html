Genesys Messenger

Bienvenido al laboratorio de integración CX de Kranon. Esta maqueta demuestra cómo administrar la visibilidad del widget de chat de Genesys dinámicamente según la ruta del usuario, permitiendo mantener una navegación limpia y dirigida.

Evaluación de Ruta
/
Habilitado (Visible)

Flujo de ejecución en esta ruta:

  1. 1
    Detección del Pathname

    El script app.js intercepta la carga de la página e identifica la ruta actual como /.

  2. 2
    Comparación contra Lista de Permitidas

    Se evalúa si la ruta está incluida en el arreglo ALLOWED_MESSENGER_PATHS, el cual contiene únicamente la raíz.

  3. 3
    Suscripción y Espera al SDK

    El código se suscribe al callback de Messenger.ready provisto por el SDK oficial de Genesys.

  4. 4
    Apertura del Launcher

    Al completarse la carga y confirmarse que la ruta está permitida, se ejecuta el comando nativo Genesys("command", "Launcher.show", {}).

Recomendación CX

Control de Parpadeo Visual (Flickering)

Para una transición fluida, configure la visibilidad del launcher en Genesys Cloud Messenger como: "Hide until triggered by business logic".

Esta directiva le indica al widget que permanezca invisible hasta que la lógica de la página ordene mostrarlo explícitamente, evitando que el chat parpadee en rutas restringidas como /nosotros o /contacto. 

Genesys("subscribe", "Messenger.ready", () => {
  // Aplicar regla de ruta
  Genesys("command", "Launcher.show", {});
});
